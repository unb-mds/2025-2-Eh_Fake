# Arquitetura C4

A documentação da arquitetura do projeto **Eh Fake** segue o modelo C4 (Context, Containers, Components and Code), permitindo uma visualização em diferentes níveis de abstração.

!!! info "Escopo da Documentação (Níveis 1 a 3)"
    Esta documentação detalha a arquitetura até o **Nível 3 (Componentes)**. 
    
    O **Nível 4 (Código)** não é representado graficamente aqui, pois reflete a implementação granular (classes, interfaces e métodos) que evolui diariamente. Para detalhes deste nível, recomenda-se a consulta direta ao código-fonte nos repositórios do GitHub.

---

## Nível 1: Contexto

Neste nível, visualizamos como o sistema **Eh Fake** se insere no ambiente, interagindo com o usuário final e dependendo de sistemas externos de Inteligência Artificial.

<div class="grid cards" markdown>

-   **Usuário**
    ---
    Pessoa que busca verificar a veracidade de uma notícia.

-   **Eh Fake (Sistema)**
    ---
    O sistema principal que orquestra a validação e exibe os resultados.

-   **Agente de IA**
    ---
    Sistema externo responsável pelo processamento de linguagem natural e classificação (RagFlow/Gemini).

</div>

## Nível 2: Contêiner

Aqui detalhamos os contêineres que compõem a plataforma, mostrando a separação entre a interface (Front End), a coleta de dados (Minerador), o processamento (IA) e o armazenamento.

!!! info "Fluxo de Dados"
    O fluxo é assíncrono: o **Minerador** coleta notícias e envia para o **Agente**, que processa e salva no **Banco Relacional**. O **Front End** apenas consulta esse banco para exibir ao usuário.

![Diagrama C4 Nível 2: Contêiner](C4 N2 Conteiner.png)

## Nível 3: Componente (Front End)

Este nível expande o contêiner **Front End** para detalhar como a interface é construída utilizando componentes React/Next.js.

!!! abstract "Responsabilidades dos Componentes"
    * **Página Principal:** Orquestrador visual.
    * **SearchBar:** Filtra notícias por título ou veracidade.
    * **NextCards:** Gerencia a navegação e paginação.
    * **OpenCard:** Exibe o detalhe completo da notícia.
    * **LoadCards:** Componente lógico que busca os dados no Banco.
